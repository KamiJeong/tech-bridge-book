# Curriculum Map: Using Claude

This map defines the intended progression from beginner readiness to practical mini projects and final safety habits.

## 00. Before You Start

- **Goal**: Set expectations and prepare a privacy-first mindset.
- **Learning outcome**: Describe what Claude is, what it is not, and what this course will help you do.
- **Key concept**: AI assistant vs. search engine vs. automation tool.
- **Work use case**: Deciding whether Claude is appropriate for an upcoming work task.
- **Path**: `ch00-before-you-start/README.md`

## 01. What Claude Is and What It Is Not

- **Goal**: Understand Claude capabilities and limits.
- **Learning outcome**: Explain Claude as a writing, thinking, research, document, and coding support assistant.
- **Key concept**: Claude as an assistant with limits, not an authority.
- **Work use case**: Introducing Claude to a colleague who has never used it.
- **Path**: `ch01-what-claude-is/README.md`

## 02. The Basic Mental Model of AI Assistants

- **Goal**: Build a simple mental model without deep technical detail.
- **Learning outcome**: Explain why Claude can be useful and still make mistakes.
- **Key concept**: Context, prediction, hallucination, and verification.
- **Work use case**: Understanding why an answer was wrong and how to correct it.
- **Path**: `ch02-mental-model/README.md`

## 03. How to Ask Better Questions

- **Goal**: Learn the ingredients of useful prompts.
- **Learning outcome**: Rewrite vague prompts into clear requests with context and format.
- **Key concept**: Role, context, task, constraint, and output format.
- **Work use case**: Turning a vague business request into a useful Claude prompt.
- **Path**: `ch03-better-questions/README.md`

## 04. Turning Messy Thoughts into Clear Prompts

- **Goal**: Practice organizing rough ideas before prompting.
- **Learning outcome**: Convert a brain dump into a structured Claude request.
- **Key concept**: Brain dump to structured prompt to refined output.
- **Work use case**: Preparing a meeting agenda from scattered notes.
- **Path**: `ch04-messy-thoughts/README.md`

## 05. Writing and Rewriting with Claude

- **Goal**: Use Claude as a writing partner for professional documents.
- **Learning outcome**: Draft, improve tone, and restructure written work with human review.
- **Key concept**: First draft, tone adjustment, structural editing.
- **Work use case**: Improving an email, report, or proposal draft.
- **Path**: `ch05-writing-rewriting/README.md`

## 06. Summarizing and Analyzing Documents

- **Goal**: Extract key information from long documents.
- **Learning outcome**: Prompt Claude for summaries, action items, and key decisions.
- **Key concept**: Document analysis, extraction, summarization.
- **Work use case**: Reviewing a long report or meeting transcript.
- **Path**: `ch06-summarizing-documents/README.md`

## 07. Research Support and Fact-Checking Habits

- **Goal**: Use Claude for research support while preserving verification habits.
- **Learning outcome**: Use Claude as a starting point and verify important claims.
- **Key concept**: Research scaffolding, source checking, hallucination risk.
- **Work use case**: Preparing a background briefing or market overview.
- **Path**: `ch07-research-support/README.md`

## 08. Planning Work with Claude

- **Goal**: Create and refine work plans with Claude.
- **Learning outcome**: Generate project plans, timelines, and checklists from a rough goal.
- **Key concept**: Work breakdown, milestones, dependencies.
- **Work use case**: Creating a product launch plan or event checklist.
- **Path**: `ch08-planning-work/README.md`

## 09. Creating Tables, Checklists, and Structured Outputs

- **Goal**: Turn unstructured input into structured outputs.
- **Learning outcome**: Ask Claude for tables, checklists, decision logs, and reports.
- **Key concept**: Structured output prompts and format constraints.
- **Work use case**: Converting meeting notes into action items.
- **Path**: `ch09-tables-checklists/README.md`

## 10. Using Claude for Multilingual Work

- **Goal**: Translate, localize, and review multilingual writing.
- **Learning outcome**: Preserve meaning and tone across languages with human review.
- **Key concept**: Translation vs. localization.
- **Work use case**: Localizing an internal announcement for global teammates.
- **Path**: `ch10-multilingual-work/README.md`

## 11. Using Claude for Technical Understanding

- **Goal**: Use Claude to understand technical topics in plain language.
- **Learning outcome**: Ask for explanations that help non-developers communicate with technical teams.
- **Key concept**: Plain-language explanation and analogy.
- **Work use case**: Understanding an error message or technical proposal.
- **Path**: `ch11-technical-understanding/README.md`

## 12. Claude for Non-Developer Collaboration with Developers

- **Goal**: Bridge business and technical communication.
- **Learning outcome**: Turn business needs into clearer developer-ready requirements.
- **Key concept**: User stories, acceptance criteria, requirement clarity.
- **Work use case**: Writing a feature brief that developers can act on.
- **Path**: `ch12-developer-collaboration/README.md`

## 13. Artifacts and Interactive Outputs

- **Goal**: Understand Claude artifacts as reusable outputs.
- **Learning outcome**: Create simple documents, tables, or interactive drafts when available.
- **Key concept**: Artifacts as shareable generated workspaces.
- **Work use case**: Creating a comparison table or formatted brief.
- **Path**: `ch13-artifacts/README.md`

## 14. Claude Code Introduction as an Advanced Path

- **Goal**: Understand when Claude Code is relevant.
- **Learning outcome**: Distinguish Claude chat from Claude Code development workflows.
- **Key concept**: Claude Code as a development workflow tool.
- **Work use case**: Deciding whether Claude Code fits a beginner developer or automation learner.
- **Path**: `ch14-claude-code-intro/README.md`

## 15. Practical Mini Projects

- **Goal**: Apply course skills through end-to-end work scenarios.
- **Learning outcome**: Complete a project from problem statement to verified output.
- **Key concept**: Multi-step prompting and verification.
- **Work use case**: Choosing a persona-based project and producing a reviewed deliverable.
- **Path**: `ch15-mini-projects/README.md`

## 16. Final Safety Checklist and Next Steps

- **Goal**: Consolidate safety habits and define a practice plan.
- **Learning outcome**: Apply the full safety checklist to any Claude task.
- **Key concept**: Privacy-first habits and human judgment.
- **Work use case**: Reviewing a Claude-assisted deliverable before sharing it.
- **Path**: `ch16-safety-next-steps/README.md`
