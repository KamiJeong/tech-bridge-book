# 실습 05: 테스트 실패 분석하기

## 목표

테스트 명령을 찾고 실패 로그를 읽어 다음 확인 단계를 정합니다.

## 준비

- 테스트가 있는 실습 저장소
- 실패 로그를 복사할 메모 공간

## 문제

테스트 실패를 보고 "어디가 틀렸는지"보다 "다음에 무엇을 확인할지"를 정합니다.

## 진행 방법

1. Codex에게 테스트 명령 후보를 찾게 합니다.
2. 빠른 테스트부터 실행합니다.
3. 실패 로그의 첫 번째 의미 있는 오류를 Codex에게 요약하게 합니다.
4. 환경 문제인지 코드 문제인지 구분할 질문을 만듭니다.

## 제출물

- 실행한 테스트 명령
- 첫 번째 실패 지점
- 다음 확인 단계
- 최종 판단

## 확인 질문

- 전체 로그보다 첫 번째 원인에 집중했나요?
- 테스트 실패를 바로 코드 문제로 단정하지 않았나요?
