# 실습 06: 변경사항 리뷰하기

## 목표

Codex가 만든 변경사항을 의도, 범위, 위험, 테스트 기준으로 리뷰합니다.

## 준비

- 변경사항이 있는 실습 브랜치
- `git diff` 확인 가능 상태

## 문제

Codex가 만든 패치를 그대로 믿지 않고 리뷰 의견을 작성합니다.

## 진행 방법

1. `git diff --stat`으로 변경 파일을 확인합니다.
2. Codex에게 diff 요약과 위험 후보를 요청합니다.
3. 파일별로 의도한 변경인지 확인합니다.
4. 리뷰 의견을 "수정 필요", "질문", "통과"로 나눕니다.

## 제출물

- 변경 파일 목록
- 위험 후보 2개
- 리뷰 의견 3개
- 추가 테스트 필요 여부

## 확인 질문

- 요청 범위 밖의 변경을 찾았나요?
- 이해하지 못한 코드를 통과시키지 않았나요?
