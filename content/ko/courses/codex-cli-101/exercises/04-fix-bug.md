# 실습 04: 작은 버그 수정하기

## 목표

버그를 재현하고, 원인 후보를 좁히고, 최소 수정으로 해결하는 흐름을 연습합니다.

## 준비

- 실패하는 테스트가 있는 실습 저장소
- 변경사항을 되돌릴 수 있는 Git 상태

## 문제

Codex에게 바로 수정을 맡기지 말고 원인 후보와 수정 계획을 먼저 받습니다.

## 진행 방법

1. 실패 메시지를 Codex에게 요약하게 합니다.
2. 원인 후보 3개와 확인 방법을 요청합니다.
3. 가장 작은 수정 계획을 승인한 뒤 수정하게 합니다.
4. `git diff`와 테스트 결과를 확인합니다.

## 제출물

- 재현 방법
- 원인 후보
- 적용한 수정 요약
- 테스트 결과

## 확인 질문

- 수정 전 계획을 확인했나요?
- 변경 범위가 버그와 직접 관련되어 있나요?
