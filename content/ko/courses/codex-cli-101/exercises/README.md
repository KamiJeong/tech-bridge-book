# Codex CLI 101 실습 목록

이 실습은 실제 회사 코드 대신 공개 예제 저장소나 개인 실습 저장소에서
진행하도록 설계되었습니다. 각 실습의 목표는 Codex에게 모든 일을 맡기는 것이
아니라, 요청하고 확인하고 리뷰하는 습관을 만드는 것입니다.

## 실습 순서

1. [Codex CLI 개념 확인](00-overview.md)
2. [설치 전 안전 점검](01-installation.md)
3. [좋은 요청문 만들기](02-basic-usage.md)
4. [낯선 코드베이스 읽기](03-read-codebase.md)
5. [작은 버그 수정하기](04-fix-bug.md)
6. [테스트 실패 분석하기](05-run-tests.md)
7. [변경사항 리뷰하기](06-review-changes.md)
8. [Spec Kit 흐름으로 나누기](07-spec-kit-workflow.md)

## 공통 규칙

- 비밀번호, API 키, 고객 정보, 회사 기밀을 사용하지 않습니다.
- Codex가 만든 변경은 반드시 `git diff`로 확인합니다.
- 이해하지 못한 코드는 병합하지 않습니다.
- 실습 결과는 "무엇을 요청했는가", "무엇을 확인했는가", "무엇을 배웠는가"로
  짧게 기록합니다.
