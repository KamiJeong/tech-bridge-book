# 실습 07: Spec Kit 흐름으로 나누기

## 목표

막연한 기능 요청을 사양, 계획, 작업 목록으로 나누는 연습을 합니다.

## 준비

- 작고 명확한 기능 아이디어 하나
- GitBook 또는 Markdown으로 메모할 공간

## 문제

"기능을 개선해줘" 같은 넓은 요청을 Codex가 구현하기 좋은 작업 단위로 바꿉니다.

## 진행 방법

1. 기능 요청을 한 문장으로 씁니다.
2. Codex에게 먼저 clarifying question을 묻게 합니다.
3. 답변을 바탕으로 spec, plan, tasks 초안을 만듭니다.
4. 작업 목록에서 가장 작은 첫 작업을 고릅니다.

## 제출물

- 기능 요청 원문
- clarifying question과 답변
- 완료 기준
- 첫 번째 작업

## 확인 질문

- 구현 전에 완료 기준을 만들었나요?
- 작업이 리뷰 가능한 크기로 나뉘었나요?
