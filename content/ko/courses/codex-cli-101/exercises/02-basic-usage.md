# 실습 02: 좋은 요청문 만들기

## 목표

목표, 범위, 제한, 검증 방법이 들어간 Codex 요청문을 작성합니다.

## 준비

- 간단한 실습 저장소
- 해결하고 싶은 작은 질문 하나

## 문제

"이거 봐줘" 같은 모호한 요청을 Codex가 실행하기 쉬운 요청으로 바꿉니다.

## 진행 방법

1. 먼저 나쁜 요청문을 한 줄로 씁니다.
2. 목표, 범위, 제한, 검증 방법을 각각 적습니다.
3. 네 요소를 합쳐 좋은 요청문으로 바꿉니다.
4. Codex에게 요청문 자체를 개선해 달라고 합니다.

## 제출물

- 나쁜 요청문 1개
- 개선한 요청문 1개
- Codex가 추가로 물어본 질문

## 확인 질문

- 수정 허용 여부를 명확히 말했나요?
- 검증 방법을 포함했나요?
