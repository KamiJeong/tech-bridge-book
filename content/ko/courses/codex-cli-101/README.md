# Codex CLI 101

Codex CLI 101은 터미널에서 동작하는 AI 코딩 에이전트를 처음 만나는
학습자를 위한 입문 강좌입니다. 목표는 명령어를 외우는 것이 아니라,
Codex에게 일을 맡길 때 무엇을 설명해야 하고, 어떤 결과를 직접 확인해야
하는지 익히는 것입니다.

## 이 강좌가 필요한 사람

- Codex CLI를 처음 설치하려는 개발자
- 낯선 코드베이스를 빠르게 파악하고 싶은 초급 개발자
- AI가 수정한 코드를 그대로 믿기보다 검토하는 습관을 만들고 싶은 사람
- Spec Kit처럼 요구사항에서 구현까지 이어지는 흐름을 배우고 싶은 사람
- 터미널은 익숙하지 않지만 AI 코딩 도구의 실제 사용 흐름을 알고 싶은 사람

## 학습 목표

이 강좌를 끝내면 다음 일을 할 수 있습니다.

- Codex CLI가 어떤 도구인지 쉬운 말로 설명한다.
- 설치와 로그인 과정에서 확인해야 할 항목을 이해한다.
- 터미널에서 Codex에게 구체적인 작업 요청을 한다.
- 낯선 코드베이스의 구조와 실행 흐름을 질문으로 파악한다.
- 작은 버그를 찾고 수정한 뒤 변경사항을 검토한다.
- 테스트를 실행하고 실패 원인을 단계적으로 좁힌다.
- AI가 만든 변경사항을 직접 리뷰하는 체크리스트를 사용한다.
- Spec Kit을 사용해 요구사항, 계획, 작업, 구현을 연결한다.

## 실습 자료

실습 자료, 예제 프롬프트, 강사용 진행 가이드는 뒤쪽 슬라이스에서 같은
코스 폴더 안에 추가됩니다.

## 배우기 전에 준비할 것

이 강좌는 실제 회사 코드나 비밀 정보가 없어도 따라 할 수 있게 구성되어
있습니다. 연습할 때는 공개 예제 저장소, 개인 실습 저장소, 또는 새로 만든
작은 프로젝트를 사용하세요.

준비물은 다음 정도면 충분합니다.

- 기본적인 터미널 실행 경험
- Git 저장소를 열어 본 경험
- 텍스트 편집기 또는 IDE
- 실습용 공개 저장소나 장난감 프로젝트

## 안전하게 사용하는 원칙

Codex CLI는 강력하지만, 최종 책임자는 사용자입니다.

- 비밀번호, API 키, 고객 정보, 회사 기밀을 프롬프트에 넣지 않습니다.
- AI가 만든 변경사항은 `git diff`로 직접 확인합니다.
- 테스트 통과만으로 충분하다고 생각하지 않고 의도와 부작용을 함께 봅니다.
- 최신 설치 방법이나 요금제 정보는 현재 공식 안내를 확인합니다.
- 이해하지 못한 코드는 병합하거나 배포하지 않습니다.

## 추천 학습 방식

각 장은 짧게 읽고 바로 따라 해보는 흐름으로 만들었습니다. 처음에는 큰
프로젝트보다 작은 저장소에서 연습하세요. Codex에게 한 번에 모든 일을
맡기기보다, "먼저 구조를 설명해줘", "수정 계획만 제안해줘", "변경사항을
검토해줘"처럼 단계를 나누는 편이 안전합니다.
