# 설치와 로그인 흐름

## 오늘 배울 것

Codex CLI를 설치하기 전에 확인할 것, 로그인 흐름에서 주의할 점, 설치 후
첫 실행에서 점검할 항목을 배웁니다.

## 왜 중요한가

AI 코딩 도구는 로컬 파일과 명령 실행 권한을 다룰 수 있습니다. 설치가
끝났다는 사실보다 어떤 계정으로 로그인했는지, 어떤 폴더에서 실행하는지,
어떤 권한을 허용했는지가 더 중요합니다.

## 쉬운 설명

설치는 보통 공식 문서의 현재 안내를 따라 진행합니다. 이 강좌에서는 특정
명령어를 외우기보다 흐름을 이해합니다.

1. 공식 설치 안내를 확인합니다.
2. 개인 또는 조직 계정으로 로그인합니다.
3. 실습용 폴더에서 Codex CLI를 실행합니다.
4. Codex가 어떤 파일과 명령을 사용할 수 있는지 확인합니다.

도구의 설치 방법, 지원 운영체제, 로그인 화면은 시간이 지나면 바뀔 수
있습니다. 그래서 오래된 블로그 글보다 현재 공식 안내를 우선해야 합니다.

## 따라 해보기

1. 공식 안내에서 설치 요구사항을 확인합니다.
2. 실습용 저장소를 준비합니다.
3. 터미널을 열고 저장소 루트로 이동합니다.
4. Codex CLI를 실행한 뒤 "현재 폴더에서 내가 조심해야 할 파일 유형을
   알려줘"라고 요청합니다.

## 흥미로운 문제

내 컴퓨터에 있는 프로젝트 중 Codex 실습에 안전한 프로젝트와 안전하지 않은
프로젝트를 나눠보세요. 고객 데이터, 회사 기밀, 개인 토큰이 들어 있을 수
있는 저장소는 실습 대상에서 제외합니다.

## Codex에게 물어볼 예제 프롬프트

```text
이 폴더에서 Codex 실습을 시작하기 전에 확인해야 할 안전 항목을
체크리스트로 만들어줘. 비밀 정보나 민감한 파일을 조심하는 관점으로 봐줘.
```

## 자주 하는 실수

- 오래된 설치 글을 보고 현재와 다른 명령을 실행하는 것
- 회사 저장소에서 바로 첫 실습을 시작하는 것
- 어떤 계정으로 로그인했는지 확인하지 않는 것
- 권한 안내를 읽지 않고 계속 진행하는 것

## 체크리스트

- [ ] 공식 설치 안내를 기준으로 삼았다.
- [ ] 실습용 저장소에서 먼저 실행했다.
- [ ] 로그인 계정과 권한 범위를 확인했다.
- [ ] 민감한 파일이 없는지 점검했다.
