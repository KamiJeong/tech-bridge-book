# 낯선 코드베이스 분석 실습

## 오늘 배울 것

처음 보는 저장소에서 Codex를 이용해 구조, 실행 흐름, 중요한 파일을 파악하는
방법을 배웁니다.

## 왜 중요한가

실무에서는 코드를 쓰는 시간보다 기존 코드를 이해하는 시간이 더 길 때가
많습니다. Codex는 검색과 요약을 도와줄 수 있지만, 질문을 잘게 나누어야
읽을 수 있는 설명이 나옵니다.

## 쉬운 설명

낯선 코드베이스를 볼 때는 세 가지를 먼저 확인합니다.

1. 이 프로젝트는 무엇을 하는가
2. 실행 시작점은 어디인가
3. 변경할 때 조심해야 할 경계는 무엇인가

Codex에게 "전체 설명"만 요구하면 너무 넓은 답이 나올 수 있습니다. 먼저
폴더 구조를 보고, 다음으로 실행 흐름을 보고, 마지막으로 특정 기능을 따라가면
좋습니다.

## 따라 해보기

1. Codex에게 주요 폴더와 파일을 요약하게 합니다.
2. `README`, 설정 파일, 테스트 폴더가 어디 있는지 물어봅니다.
3. 한 기능을 골라 "요청이 들어와서 응답이 나가기까지 어떤 파일을 거치는지"
   설명해 달라고 합니다.
4. Codex 설명과 실제 파일을 번갈아 확인합니다.

## 흥미로운 문제

Codex가 추천한 "먼저 읽을 파일" 목록에서 하나를 빼고 다른 파일을 넣어야
한다면 무엇을 고르겠습니까? 이유를 적어 보세요.

## Codex에게 물어볼 예제 프롬프트

```text
이 저장소를 처음 맡은 초급 개발자에게 설명한다고 생각해줘.
1. 주요 폴더
2. 실행 시작점
3. 테스트 위치
4. 수정할 때 조심할 부분
을 짧게 정리해줘. 확실하지 않은 내용은 확인 방법도 같이 알려줘.
```

## 자주 하는 실수

- 파일 이름만 보고 역할을 단정하는 것
- Codex 요약을 실제 코드 확인 없이 믿는 것
- 테스트와 설정 파일을 읽지 않고 기능 파일만 보는 것
- 큰 프로젝트 전체를 한 번에 설명해 달라고 하는 것

## 체크리스트

- [ ] 프로젝트 목적을 한 문장으로 설명할 수 있다.
- [ ] 실행 시작점과 테스트 위치를 찾았다.
- [ ] Codex 요약과 실제 파일을 대조했다.
- [ ] 모르는 부분을 작은 질문으로 나누었다.
