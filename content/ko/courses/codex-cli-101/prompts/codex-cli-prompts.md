# Codex CLI 예제 프롬프트 모음

이 프롬프트는 그대로 복사하기보다 상황에 맞게 바꿔 쓰는 출발점입니다.

## 분석만 요청하기

```text
이 저장소를 처음 보는 개발자에게 설명한다고 생각해줘.
아직 파일을 수정하지 말고 주요 폴더, 실행 시작점, 테스트 위치만 정리해줘.
확실하지 않은 내용은 추측이라고 표시해줘.
```

## 수정 전 계획 받기

```text
이 문제를 바로 수정하지 말고 먼저 계획만 세워줘.
원인 후보, 확인할 파일, 실행할 명령, 최소 수정 방향을 순서대로 알려줘.
내가 승인하기 전까지 파일은 수정하지 마.
```

## 테스트 명령 찾기

```text
이 저장소에서 실행 가능한 테스트 명령을 찾아줘.
각 명령이 확인하는 범위와 예상 소요 시간을 설명하고, 빠른 확인용 명령을
먼저 추천해줘.
```

## 실패 로그 해석하기

```text
아래 테스트 실패 로그를 요약해줘.
첫 번째 의미 있는 실패 지점, 가능한 원인, 다음에 확인할 명령을 나눠서 알려줘.
환경 문제일 가능성도 따로 표시해줘.
```

## 변경사항 리뷰하기

```text
현재 git diff를 리뷰해줘.
요약, 요청 범위와 맞는 변경, 의도하지 않았을 수 있는 변경, 테스트 필요 항목,
보안이나 개인정보 위험을 나눠서 알려줘.
```

## Spec Kit으로 정리하기

```text
이 요청을 바로 구현하지 말고 Spec Kit 흐름으로 정리해줘.
먼저 사용자, 목표, 범위, 완료 기준, 위험한 가정을 질문해줘.
답을 받은 뒤 spec, plan, tasks 순서로 나눠줘.
```

## 안전 점검하기

```text
이 작업을 Codex에게 맡기기 전에 안전 체크리스트를 만들어줘.
비밀 정보, 고객 데이터, 회사 기밀, 되돌리기 어려운 변경, 테스트 필요 항목을
중심으로 봐줘.
```
