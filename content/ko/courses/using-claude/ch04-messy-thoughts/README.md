# 흩어진 생각을 명확한 프롬프트로 바꾸기

이 문서는 GitBook에서 Claude 과정을 탐색하고 작성할 때 사용하는 기준 문서입니다.

## 목표

프롬프트를 쓰기 전에 생각을 정리하는 습관을 연습합니다.

## 학습 결과

메모와 생각의 덩어리를 구조화된 Claude 요청으로 바꿀 수 있습니다.

## 핵심 개념

생각 쏟아내기에서 구조화된 프롬프트와 개선된 출력으로 이어지는 흐름.

## 실무 활용

흩어진 메모로 회의 안건 만들기.

## 레슨 작성 자리표시자

이 챕터는 현재 과정 구조만 정의합니다. 전체 레슨을 작성할 때는 과정 레슨 템플릿을 사용하고 모든 필수 섹션을 포함해야 합니다.

- 제목
- Who this lesson is for / 이 레슨의 대상
- 배울 내용
- 실무에서 중요한 이유
- Core concept explanation / 핵심 개념 설명
- Simple example / 쉬운 예시
- Claude 프롬프트 예시
- 실습
- 생각해 볼 질문
- 자주 하는 실수
- 안전 체크리스트
- 요약
- 다음 단계

## 안전 안내

기능 제공 여부는 Claude 요금제와 현재 인터페이스에 따라 달라질 수 있습니다. 항상 본인의 Claude 화면에서 확인하세요.

공식 업무에 사용하기 전에는 사람이 사실, 맥락, 표현, 민감정보 포함 여부를 검토해야 합니다.
